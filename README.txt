# Descriptive Project Repo

Repo ini adalah simulasi kolaborasi dalam sebuah project git

## Cara kerja

Setiap file markdown berisikan paragraf deskriptif yang menjelaskan tentang sebuah fitur, misal:

```plain
main.md

variabel a
variabel b

## fitur 1
Fitur 1 ini berguna untuk kalkulasi antara variabel a dan b, variabel a dan b akan dijumlahkan lalu dibagi 2.

## fitur 1a
Hasil fitur 1 akan di tentukan apakah (a+b) >= 5 ?, jika ia hasilkan `true`, sebaliknya hasilkan `false`

```

Ini akan membuat simulasi sebuah program tanpa mempelajari satupun bahasa pemrograman.

## Tujuan

Maintainer serta kontributor diharapkan akan menguasai konsep merging dalam git dan github. Dimulai dari merging semua pull request yang akan datang sampai hanya merging bagian/kode tertentu dari pull request yang akan datang.
